## 平台设计总结

### 核心原则
1. **一切皆 Markdown**: 所有文档都是 AI 可读的 Skill
2. **Skill 驱动**: AI 根据 Skill 描述执行
3. **无需代码实现**: AI 自己就是"引擎"

### 工作原理
```
用户需求 → AI 读取 Skill → AI 执行步骤 → AI 记录状态
```

### Skill 类型
1. **平台 Skill** (`skills/PLATFORM.md`)
   - 标准开发流程（5 个步骤）
   - 项目识别规则
   - AI 执行约定

2. **项目 Skill** (`skills/projects/*.md`)
   - 技术栈
   - 架构设计
   - 编码规范
   - 开发偏好

## 关键学习

### Skill 驱动的优势

1. **无需代码实现**
   - 传统方式：需要写状态机、流程引擎
   - Skill 方式：AI 直接理解执行

2. **易于维护**
   - 修改流程只需更新 Markdown
   - 不需要修改代码和重新部署

3. **AI 友好**
   - Markdown 是 AI 训练数据的常见格式
   - AI 能理解标题、列表、表格、代码块

4. **人类友好**
   - 任何人都能编辑 Markdown
   - 无需编程知识
